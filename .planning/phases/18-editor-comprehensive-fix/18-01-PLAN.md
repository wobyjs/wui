# Phase 18 Wave 1: Core Selection Infrastructure

**Phase**: 18
**Wave**: 1 of 3
**Status**: Ready

## Goal

Fix the shadow-DOM selection blindspot that is the root cause of every formatting bug in this editor. After this wave, every piece of code that reads or saves the active selection will see the real text nodes inside the `<wui-editor>` shadow root instead of the host element boundary. Computed style detection will also be fixed so that semantic elements (`<strong>`, `<em>`, `<b>`) are recognized correctly, and cross-block selection offsets will work via editor-root-relative character counting.

Decisions addressed: D-01, D-02, D-04, D-11.

## Prerequisites

None. This is Wave 1.

### Task 1.1: BrowserCompat — shadow-DOM-aware safeGetRange (D-01, D-11)

**File**: `src/Editor/BrowserCompat.ts`

**Decision**: D-01, D-11

**Description**:
`safeGetRange()` currently calls `sel.getRangeAt(0)` unconditionally (line 74). For content inside a shadow root, this returns a range ending at the host element boundary, not at the actual text node. The fix adds an optional `shadowRoot` parameter and, when provided, uses `Selection.getComposedRanges({shadowRoots:[shadowRoot]})` (Chrome 137+, Firefox 142+, Safari 17+). Because `getComposedRanges` returns `StaticRange` objects and `selection.addRange()` only accepts a live `Range`, a `staticRangeToLiveRange` helper must clone the result before returning it.

`FocusManager.cacheSelection()` (line 93 in `FocusManager.ts`) calls `sel.getRangeAt(0)` directly — this must be updated in the same wave as `safeGetRange` or the cached path will still be shadow-root-blind (Risk 4 in RESEARCH.md). The fix: replace the direct call with `safeGetRange(editorEl.getRootNode() as ShadowRoot)`.

`getSelectionInfo()` in BrowserCompat.ts (line 174) also calls `sel.getRangeAt(0)` directly — update it to call `safeGetRange(root.getRootNode() as ShadowRoot)` using the `root` parameter it already receives.

**Implementation**:

Add `staticRangeToLiveRange` helper immediately above `safeGetRange`, then replace the entire `safeGetRange` function (lines 51-75):

```typescript
/**
 * Clone a StaticRange (from getComposedRanges) to a live Range.
 * selection.addRange() only accepts live Range — StaticRange is not accepted.
 */
function staticRangeToLiveRange(sr: StaticRange): Range {
    const range = document.createRange()
    range.setStart(sr.startContainer, sr.startOffset)
    range.setEnd(sr.endContainer, sr.endOffset)
    return range
}

/**
 * Get the first range from selection, shadow-DOM-aware.
 * Pass shadowRoot to pierce the shadow boundary via getComposedRanges.
 * Falls back to getRangeAt(0) for same-document usage or older browsers.
 *
 * D-01: shadow root must be passed by all callers operating inside wui-editor.
 */
export function safeGetRange(shadowRoot?: ShadowRoot): Range | null {
    const sel = window.getSelection()
    if (!sel) return null

    // Path 1: shadow-DOM-aware (Chrome 137+, Firefox 142+, Safari 17+)
    if (shadowRoot && typeof (sel as any).getComposedRanges === 'function') {
        const composed: StaticRange[] = (sel as any).getComposedRanges({ shadowRoots: [shadowRoot] })
        if (composed.length > 0) {
            return staticRangeToLiveRange(composed[0])
        }
        return null
    }

    // Dev-mode warning when shadow root provided but API unavailable
    if (shadowRoot && typeof (sel as any).getComposedRanges !== 'function') {
        if (typeof process !== 'undefined' && process.env?.NODE_ENV !== 'production') {
            console.warn('[BrowserCompat] getComposedRanges not supported; shadow DOM selection may be inaccurate. Requires Chrome 137+, Firefox 142+, Safari 17+.')
        }
    }

    // Path 2: rangeCount=0 recovery for backspace/delete edge cases (kept intact)
    if (sel.rangeCount === 0) {
        if (sel.focusNode) {
            try {
                const range = document.createRange()
                range.setStart(sel.focusNode, sel.focusOffset)
                range.collapse(true)
                return range
            } catch (e) {
                return null
            }
        }
        return null
    }

    return sel.getRangeAt(0)
}
```

### Task 1.2: StyleEngine — fix getComputedStyles for semantic elements (D-02)

**File**: `src/Editor/StyleEngine.ts`

**Decision**: D-02

**Description**:
`getComputedStyles()` (lines 17-25) returns `node.style` — the element's inline `style` attribute only. This means `<strong>`, `<em>`, `<b>`, `<i>` are never detected as bold/italic because they carry no inline style; their rendered boldness comes from the UA stylesheet. Replace with `window.getComputedStyle(node)` which resolves all CSS layers including UA defaults.

Critical cross-browser pitfall: `getComputedStyle(el).textDecoration` returns the shorthand `"underline solid rgb(0,0,0)"` in Chrome, not `"underline"`. Must use `text-decoration-line` instead. The functions `applyUnderline()` and `applyStrikethrough()` currently call `applyStyle('textDecoration', 'underline')` and `applyStyle('textDecoration', 'line-through')` — change both to use `'textDecorationLine'` so detection and application use the same property name.

The existing `normalizeFontWeight()` already handles `"bold"` vs `"700"` — no changes needed there.

The existing `toggleStyle()` logic at lines 854-895 is structurally correct for D-03 semantics (all-styled -> remove, any-unstyled -> apply). After D-02 lands, it will work correctly for semantic elements without any logic changes (per RESEARCH.md D-03 section).

**Implementation**:

Replace `getComputedStyles` (lines 17-25):

```typescript
/**
 * Get computed styles from a node, including UA stylesheet and inherited values.
 * D-02: window.getComputedStyle resolves <strong>, <em>, <b>, <i> correctly.
 * node.style only returned inline styles — semantic elements were invisible.
 */
function getComputedStyles(node: Node): CSSStyleDeclaration {
    if (node instanceof HTMLElement) {
        return window.getComputedStyle(node)
    }
    if (node instanceof Text && node.parentElement) {
        return window.getComputedStyle(node.parentElement)
    }
    return {} as CSSStyleDeclaration
}
```

Update `applyUnderline()` to use `textDecorationLine`:

```typescript
export function applyUnderline(): void {
    applyStyle('textDecorationLine', 'underline')
}
```

Update `applyStrikethrough()` to use `textDecorationLine`:

```typescript
export function applyStrikethrough(): void {
    applyStyle('textDecorationLine', 'line-through')
}
```

### Task 1.3: StyleEngine — editor-root-relative character offsets for cross-block selection (D-04)

**File**: `src/Editor/StyleEngine.ts`

**Decision**: D-04

**Description**:
`saveSelectionAsOffsets()` (lines 167-199) calls `getBlockParent(range.commonAncestorContainer)` as the anchor. For cross-block selections, `range.commonAncestorContainer` is the editor `<div>` — not in `BLOCK_TAGS` — so `getBlockParent()` returns `null`, and selection restoration is entirely skipped for every cross-paragraph operation.

Replace `saveSelectionAsOffsets` and `restoreSelectionFromOffsets` with editor-root-relative implementations using a `findEditorRoot()` helper. These walk the entire text tree from `[data-editor-root]` and accumulate global character offsets — these offsets survive span wrapping/unwrapping because total text content is preserved.

Update every call site (in `applyStyle`, `removeStyle`, `toggleStyle`, `applyTextAlign`, `applyIndent`, `applyList`) to use the new `{ editorRoot, startOffset, endOffset }` shape. Update the `normalizeDOM` call sites in `applyStyle` and `removeStyle` to fall back to `findEditorRoot` when `getBlockParent` returns null (cross-block ranges).

**Implementation**:

Add `findEditorRoot()` helper after `getBlockParent()`:

```typescript
/**
 * Walk up to the [data-editor-root] element.
 * D-04: used as stable anchor for global character offset calculations.
 */
function findEditorRoot(node: Node): HTMLElement | null {
    let current: Node | null = node
    while (current) {
        if (current instanceof HTMLElement && current.dataset && 'editorRoot' in current.dataset) {
            return current
        }
        current = current.parentNode
    }
    return null
}
```

Replace `saveSelectionAsOffsets` (lines 167-199) with:

```typescript
/**
 * Save selection as global character offsets from the editor root.
 * D-04: editor-root-relative offsets survive cross-block selections and span wrapping.
 * Returns { editorRoot: null } if editor root cannot be found.
 */
function saveSelectionAsOffsets(range: Range): {
    editorRoot: HTMLElement | null
    startOffset: number
    endOffset: number
} {
    const editorRoot = findEditorRoot(range.commonAncestorContainer)
    if (!editorRoot) return { editorRoot: null, startOffset: 0, endOffset: 0 }

    const walker = document.createTreeWalker(editorRoot, NodeFilter.SHOW_TEXT, null)
    let currentOffset = 0
    let startOffset = -1
    let endOffset = -1
    let node: Node | null

    while ((node = walker.nextNode())) {
        const textNode = node as Text
        const len = textNode.textContent?.length ?? 0

        if (startOffset === -1 && node === range.startContainer) {
            startOffset = currentOffset + range.startOffset
        }
        if (endOffset === -1 && node === range.endContainer) {
            endOffset = currentOffset + range.endOffset
        }

        currentOffset += len
        if (startOffset !== -1 && endOffset !== -1) break
    }

    if (startOffset === -1) startOffset = 0
    if (endOffset === -1) endOffset = startOffset

    return { editorRoot, startOffset, endOffset }
}
```

Replace `restoreSelectionFromOffsets` (lines 204-277) with:

```typescript
/**
 * Restore selection from global character offsets saved by saveSelectionAsOffsets.
 * D-04: walks from the editor root, not from a block parent.
 */
function restoreSelectionFromOffsets(
    editorRoot: HTMLElement,
    startOffset: number,
    endOffset: number
): void {
    const sel = safeGetSelection()
    if (!sel) return

    const walker = document.createTreeWalker(editorRoot, NodeFilter.SHOW_TEXT, null)
    let currentOffset = 0
    let startNode: Text | null = null
    let startNodeOffset = 0
    let endNode: Text | null = null
    let endNodeOffset = 0
    let node: Node | null

    while ((node = walker.nextNode())) {
        const textNode = node as Text
        const len = textNode.textContent?.length ?? 0

        if (startNode === null && startOffset >= currentOffset && startOffset <= currentOffset + len) {
            startNode = textNode
            startNodeOffset = startOffset - currentOffset
        }
        if (endNode === null && endOffset >= currentOffset && endOffset <= currentOffset + len) {
            endNode = textNode
            endNodeOffset = endOffset - currentOffset
        }

        currentOffset += len
        if (startNode !== null && endNode !== null) break
    }

    if (startNode && endNode) {
        try {
            const newRange = document.createRange()
            newRange.setStart(startNode, startNodeOffset)
            newRange.setEnd(endNode, endNodeOffset)
            sel.removeAllRanges()
            sel.addRange(newRange)
        } catch (e) {
            console.warn('[restoreSelectionFromOffsets] Failed to restore selection:', e)
        }
    } else {
        console.warn('[restoreSelectionFromOffsets] Could not find nodes for offsets:', { startOffset, endOffset, startNodeFound: !!startNode, endNodeFound: !!endNode })
    }
}
```
