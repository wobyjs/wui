    1. **FocusManagerContext** - Woby context for sharing focus state across components
       ```typescript
       interface FocusManager {
         isFocused: Observable<boolean>
         saveSelection(): void
         restoreSelection(): boolean
         clearSelection(): void
         registerToolbar(element: HTMLElement): () => void
         registerEditor(element: HTMLElement): () => void
       }
       ```

    2. **SelectionCache class** - Private class for caching selection state
       - Use `saveSelectionAsOffsets` pattern from StyleEngine.ts (lines 167-199)
       - Store: `{ block: HTMLElement | null, startOffset: number, endOffset: number }`
       - Method `save()`: Save current selection using `safeGetSelection()` and `safeGetRange()`
       - Method `restore()`: Restore selection using `restoreSelectionFromOffsets()` pattern
       - Method `clear()`: Clear cached selection

    3. **Focus tracking** - Track editor focus state
       - `isFocused: Observable<boolean>` - reactive focus state
       - `registerEditor(element)`: Attach focus/blur listeners to editor element
       - On focus: `isFocused(true)`
       - On blur: Check if focus moved to registered toolbar (use `relatedTarget`), if not `isFocused(false)`

    4. **Toolbar coordination** - Setup capture-phase listeners on toolbar
       - `registerToolbar(element)`: Attach capture-phase mousedown listener
       - On mousedown (capture phase): Call `saveSelection()` BEFORE focus shift
       - Return cleanup function to remove listener

    5. **Context provider** - `FocusManagerProvider` component
       - Create FocusManager instance
       - Provide via `FocusManagerContext.Provider`
       - Accept `editorRef` prop for editor element registration

    Implementation notes:
    - Use `useEffect` for attaching event listeners (Woby's reactive lifecycle)
    - Use `{ capture: true }` for mousedown listener to intercept before Woby's delegation
    - Import `safeGetSelection`, `safeGetRange` from './BrowserCompat'
    - DO NOT call `e.preventDefault()` in FocusManager - that's the toolbar button's responsibility
    - FocusManager only caches selection, does not prevent focus shift
