Change isEditing default and toolbar visibility to use FocusManager state, making the editor immediately usable.

Purpose: Currently the editor requires clicking to activate (isEditing starts false, contentEditable bound to isEditing). This is a poor UX — users expect to click into an editor and start typing immediately. Wire FocusManager.isFocused to control toolbar visibility and make contentEditable always true.

Output: Editor that is immediately editable with toolbar that shows/hides based on focus state from FocusManager.
</objective>


       ```typescript

       ```

    5. **Ensure FocusManager blur handling is correct**:
       The FocusManager's `registerEditor` method handles blur by checking if focus moved to a registered toolbar element. When focus leaves the editor AND toolbar (e.g., clicking outside), `focusManager.isFocused` becomes false, hiding the toolbar. When focus is on the toolbar (e.g., clicking a button), `focusManager.isFocused` stays true, keeping the toolbar visible.

    6. **Remove any `tabIndex` workaround** from the editor div if present. The editor is now always contentEditable, so it naturally receives focus on click.

    NOTE: If `isEditing` is used elsewhere in the component (e.g., for placeholder text), replace those uses with `focusManager.isFocused()` as well.
