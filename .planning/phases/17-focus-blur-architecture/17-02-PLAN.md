Output: All toolbar buttons integrated with FocusManager, Editor wrapped with FocusManagerContext.Provider.

    1. Import `useFocusManager`:
       ```typescript

       ```


    3. Replace the `onClick` handler to restore selection before applying format:
       Before:
       ```jsx

       ```
       After:
       ```jsx
       <Button
         onMouseDown={(e) => { e.preventDefault(); }}

         }}
       ```

    **Keep the existing `onMouseDown={(e) => { e.preventDefault(); }}`** on each button. This provides a belt-and-suspenders approach:
    - The toolbar-level capture-phase listener (Plan 01) saves selection BEFORE focus shift
    - The individual `e.preventDefault()` on each button is a secondary defense
    - The `restoreSelection()` in onClick ensures selection is present for the command

    **Pattern for dropdown components** (TextFormatDropDown, TextFormatOptionsDropDown, FontFamilyDropDown, TextAlignDropDown, FontSize, InsertDropDown):

    These use `onMouseDown={(e) => { e.preventDefault(); }}` already. Keep that. For the dropdown option click handlers, add `focusManager.restoreSelection()` at the start of each option's action handler, before the style/format command.

    Example for TextFormatDropDown:
    ```typescript
    const handleHeadingSelect = (tag: string) => {
      focusManager.restoreSelection()
      applyBlockFormat(editor, tag)
    }
    ```

    **Pattern for color pickers** (TextColorPicker, TextBackgroundColorPicker):

    These have `onMouseDown={(e) => { e.preventDefault(); }}` and an `onChange` handler on a color input. Add `focusManager.restoreSelection()` at the start of the onChange handler:


    **Pattern for UndoRedoButton**:

    Add `focusManager.restoreSelection()` before the undo/redo action. Undo/redo may change the DOM, so restoring selection before the action ensures the operation targets the correct range.

    Same pattern — add `focusManager.restoreSelection()` at the start of the onClick handler, before the list/indent/align command.

    IMPORTANT: Do NOT remove the `onMouseDown={(e) => { e.preventDefault(); }}` from any button. It serves as a secondary defense even though Woby's delegation may limit its effectiveness. The capture-phase listener on the toolbar container is the primary fix.
