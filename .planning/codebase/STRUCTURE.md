## Directory Purposes

**`src/`:**
- Purpose: All source code for the component library
- Contains: React/JSX components, effects, utilities
- Key files: `index.tsx`, `main.ts`, `app.tsx`

**`src/Editor/`:**
- Purpose: Rich text editor subsystem
- Contains: `Editor.tsx`, toolbar components, formatting utilities
- Key files: `Editor.tsx` (main), `utils.tsx` (DOM utilities), `undoredo.tsx` (history context)

**`src/Wheeler/`:**
- Purpose: Picker/wheel selector components
- Contains: `Wheeler.tsx` (main picker), `DateTimeWheeler.tsx`, `MultiWheeler.tsx`
- Key files: `WheelerType.tsx` (shared types), `Wheeler.tsx` (105KB, complex picker logic)

**`src/PropertyForm/`:**
- Purpose: Dynamic property editor for objects
- Contains: PropertyForm, individual editors (Boolean, String, Number, Color, etc.)
- Key files: `PropertyForm.tsx`, `DropdownEditor.tsx`

**`src/icons/`:**
- Purpose: SVG icon components
- Contains: 27 icon files (bold.tsx, italic.tsx, h1.tsx, etc.)
- Pattern: Each file exports one icon component

**`src/helper/`:**
- Purpose: Shared utilities and code blocks
- Contains: `helper.tsx`, `CodeBlock.tsx`

**`test/`:**
- Purpose: Component tests
- Contains: `.test.tsx` and `.test.html` pairs for each component
- Structure: Mirror of `src/` layout

**`docs/`:**
- Purpose: Documentation site
- Contains: Markdown docs, API references, guides
- Key files: `docs/index.tsx` (doc site renderer)

**`dist/`:**
- Purpose: Build output (DO NOT edit manually)
- Contains: Compiled JS, CSS, TypeScript declarations
- Generated by: `pnpm run build`

## Key File Locations

**Entry Points:**
- `src/index.tsx`: Library barrel export — what consumers import
- `src/main.ts`: Development server entry — renders demo app
- `vite.config.mts`: Build entry config — controls what gets compiled

**Configuration:**
- `package.json`: Dependencies, scripts, exports
- `tsconfig.json`: TypeScript compiler options
- `vite.config.mts`: Vite/Rollup build configuration

**Core Logic:**
- `src/Button.tsx`: Button with variant styles and custom element registration
- `src/TextField.tsx`: Text input with effects and adornments
- `src/Switch.tsx`: Toggle switch with effect presets
- `src/Editor/Editor.tsx`: Rich text editor with toolbar
- `src/Wheeler/Wheeler.tsx`: Scroll picker with touch support
- `src/PropertyForm/PropertyForm.tsx`: Dynamic object property editor

**Testing:**
- `test/TextField.test.tsx`: Unit test for TextField
- `test/Editor/List.test.tsx`: Unit test for Editor List feature
- `test/Wheeler/Wheeler.test.tsx`: Unit test for Wheeler component

## Naming Conventions

**Files:**
- PascalCase: Component files (`Button.tsx`, `TextField.tsx`, `Editor.tsx`)
- camelCase: Effect files (`TextField.effect.tsx`, `Switch.effect.tsx`)
- kebab-case: Icon files (`align_center.tsx`, `list_bulleted.tsx`)

**Components:**
- PascalCase: Exported component functions (`TextField`, `Button`)
- camelCase: Internal helper components (`StartAdornment`, `EditorSurface`)
- prefixed with module name: Button variants (`variant.text`, `variant.contained`)

**Directories:**
- PascalCase: Subsystem directories (`Editor/`, `Wheeler/`, `PropertyForm/`)
- lowercase: Utility directories (`icons/`, `helper/`)

## Where to Add New Code

**New Component (top-level):**
- Primary: Create `src/NewComponent.tsx`
- Effects: Create `src/NewComponent.effect.tsx` for styling presets
- Tests: Create `test/NewComponent.test.tsx` and `test/NewComponent.test.html`

**New Editor Toolbar Button:**
- Implementation: Create `src/Editor/FormatButton.tsx`
- Register in `src/Editor/Editor.tsx` toolbar groups

**New Wheeler Picker Variant:**
- Implementation: Create `src/Wheeler/NewWheeler.tsx` or extend `Wheeler.tsx`
- Types: Update `src/Wheeler/WheelerType.tsx`

**New Property Editor:**
- Implementation: Create `src/PropertyForm/NewEditor.tsx`
- Register in `src/PropertyForm/PropertyForm.tsx` Editors array

**New Icon:**
- Implementation: Create `src/icons/icon_name.tsx`
- No test needed (pure presentational)
